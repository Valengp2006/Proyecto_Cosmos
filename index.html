Concepto

Un diálogo
a tres bandas

Cosmos propone una conversación entre sonido, imagen y presencia humana. La obra no es fija: respira, cambia de color, colapsa hacia el vórtice o se expande en estrellas.

Cada espectador se convierte en autor. Desde su propio dispositivo, tiñe la Luna con colores en tiempo real. El performer guía la tensión narrativa. El audio sincroniza el parpadeo.

Es una exploración de cómo las interfaces web pueden difuminar la barrera entre espectador y obra.

Capa 01
Pulso Sonoro
Live coding en Strudel. El kick drum sincroniza el parpadeo lunar via OSC.
Capa 02
Performer
Controla los estados visuales desde iPad. Estrellas → Vórtice → Nebulosa.
Capa 03
Público Activo
Datos RGB en tiempo real desde smartphones. El color de la Luna es colectivo.

Arquitectura

Red local
distribuida

OSC + WebSockets garantizan comunicación simultánea y de baja latencia entre todos los nodos del ecosistema.

Componente Plataforma Puerto Función
Audio (Pulsar) Strudel + OSC Bridge 8080 UDP Sincroniza parpadeo lunar con el kick drum bd
Performer iPad + Open Stage Control 9000 UDP Controla transiciones de estados y ciclo evolutivo
Público Smartphones + Node.js / Socket.io 7000 UDP Envía color RGB en tiempo real → tiñe la Luna
Servidor Web Node.js 3000 HTTP Interfaz del espectador vía QR

Operación

Cómo ejecutar
la obra

01
Preparación de Red
Activar el Hotspot móvil del dispositivo host. Conectar MacBook Pro e iPad a esta red. Obtener la IP local del host (ej. 172.20.10.7).
02
Audio
Cargar el script de /Audio en Strudel. Verificar salida de paquetes OSC hacia el puerto 8080.
03
Visuales
Abrir el .toe en TouchDesigner. Confirmar que los tres nodos receptores TDStrudelSync, oscin1 y oscin2 estén activos en sus puertos.
04
Performer
Iniciar Open Stage Control con port: 8082, send: 127.0.0.1:9000. Abrir la interfaz en el iPad.
05
Público
En la terminal: navegar a /Servidor y ejecutar node server.js. El público accede escaneando el QR apuntando a http://[IP]:3000.
